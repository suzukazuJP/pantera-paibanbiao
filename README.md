- 公開URL（従業員）: https://suzukazujp.github.io/pantera-paibanbiao/
- 管理控制台: 上記URL + `/admin.html`
- 商家ページ: 上記URL + `/shop.html`

## 📁 フォルダは2つある（役割が違う）

1. **【本番】`書類/GitHub/pantera-paibanbiao`** ← 🟢 これが本物
   - GitHub と繋がっている。ここを編集して push すると公開サイトに反映される。
2. **【コピー】`デスクトップ/20260612paibanbiao`** ← 🟡 ただの作業用コピー
   - 本番と同じ内容を手でコピーして保っているだけ。

**迷ったら必ず「本番（GitHub側）」を正しいものとして扱う。**

---

## 🚀 更新（公開）のしかた

1. ファイルを編集する
2. **GitHub Desktop** を開く
3. 変更内容を確認して **Commit to main**
4. **Push origin** を押す
5. 数分で公開URLに反映される

---

## 💾 データ（ユーザー登録・シフト・ログ）について

ユーザーやシフトのデータは、**これらのHTMLファイルの中ではなく Firebase（クラウド）に保存**されています。

- ✅ HTMLファイルを消しても、登録済みデータは消えません（クラウドに残る）
- ⚠️ ただしHTMLが本体なので、**ファイルを消すとアプリの画面が無くなります**

---

## 🛡️ バックアップの安心ポイント

- **GitHub に push してあれば、パソコンのファイルが消えても GitHub から元に戻せます。**
- だから **「編集したらこまめに Push」** が、一番かんたんで確実なバックアップです。
- さらに安心したい場合は、このフォルダごと外付けドライブやクラウド（iCloud / Google Drive 等）にコピーしておくと二重で安全です。
